# 🧭 통합 업무 자동화 플랫폼 기획안

**구성:** Discord 업무 서버 + Notion 업무 현황 DB + 자동화 봇 시스템

**우선순위:** 소통 ＞ 의사결정 ＞ 알림 ＞ 지식축적

---

## Ⅰ. 전체 구조 개요

### 🎯 목표

- Discord를 **실시간 협업 허브**로,
    
    Notion을 **업무 데이터베이스 및 문서 저장소**로 사용.
    
- 봇을 통해 **상태 자동 동기화 / 알림 자동화 / 회의록 백업** 구현.

### 🧩 시스템 구성도

```
[User Actions]
   ↓
[Discord Channels]  ← 실시간 커뮤니케이션, 명령 입력
   ↓
[Automation Bot]    ← 업무 등록, 상태 변경, 회의록 전송
   ↓
[Notion Database]   ← 업무 현황, 회의록, 인시던트 기록 저장
   ↑
[Google Calendar / GitHub / Forms] ← 외부 일정·이슈 연동

```

## Ⅳ. Discord ↔ Notion 자동화 흐름

### 🔄 1. 신규 업무 등록

- **Trigger:**
    
    사용자가 Discord에서 `/task add [내용] [팀] [담당자]` 명령 입력.
    
- **자동화 동작:**
    1. 봇이 메시지를 Notion DB에 `Task Name`, `팀`, `담당자`, `상태=To Do` 로 등록.
    2. 메시지 링크 자동 삽입.
    3. 봇이 Discord에서 “업무가 등록되었습니다 ✅” 알림 전송.

```
예시:
/task add “멘토링톤 4차시 교안 검수” 팀:교육기획 담당자:@현중
→ Notion: Task 등록 → Discord: #교육기획-general 에 자동 안내

```

---

### 🔁 2. 상태 업데이트

- **Trigger:**
    
    `/task done [업무명]`
    
- **동작:**
    - 봇이 Notion DB에서 해당 Task 찾아 `Status = Done` 으로 변경.
    - Discord 채널에 완료 메시지 전송.
    - `#calendar` 채널에 “업무 완료” 자동 알림.

---

### 🗓️ 3. 일정 연동 (Google Calendar)

- **Trigger:**
    
    Notion Task에 마감일 입력 시 → Google Calendar에 자동 이벤트 생성.
    
- **동작:**
    - 봇이 일정 제목과 담당자를 읽어 Discord `#calendar` 채널에 일정 생성 알림 전송.

---

### 🧾 4. 회의록 자동 생성 및 백업

- **Trigger:**
    
    `/meeting note [제목]`
    
- **동작:**
    - 봇이 `#meeting-notes`에 회의록 템플릿 메시지 생성.
    - 회의 종료 후 `/meeting save` 입력 시 → 내용 자동 요약 → Notion “회의록 DB” 저장.
    - 해당 회의록 URL이 자동으로 관련 Task의 “회의록 링크” 필드에 추가됨.

---

### 🚨 5. 인시던트 자동 관리

- **Trigger:**
    
    `#incident-report`에 메시지 등록 시.
    
- **동작:**
    - 봇이 메시지를 Notion “이슈 관리 DB”에 자동 기록.
    - 심각도(🔥 / ⚠️ 이모지 반응) 기반으로 필드 자동 분류.

## Ⅴ. 자동화 봇 시스템 설계

### ⚙️ 기술 스택

| 구성 요소 | 기술 | 설명 |
| --- | --- | --- |
| Backend | Python (FastAPI) | Discord 명령 처리 및 Notion API 호출 |
| Discord Bot | `discord.py` | 메시지 감지, 명령 실행 |
| Notion API | `notion-client` | DB 업데이트 및 조회 |
| Database | SQLite / PostgreSQL | Discord–Notion 매핑 캐시 저장 |
| Scheduler | APScheduler | 일일 리마인더, 미완료 업무 알림 |

## Ⅶ. 보안 및 접근제어

- Discord 모든 사용자 **2FA 필수**
- @everyone 메시지 전송 제한
- 외부자(협력사) 카테고리 별도 운영
- Notion API Key → AWS KMS로 암호화
- Bot Token은 `.env` 파일에 저장
- Task 수정권한: 담당자와 팀장만
